CMS Architecture (current state as of 6/1/2026)


1. Request flow (what happens when a page loads)

	Browser Request
		  ↓
	public/index.php
		  ↓
	Router::dispatch($route)
		  ↓
	ServiceContainer::get('cms')
		  ↓
	ContentCMSService::load($route)
		  ↓
	ContentCMS::loadPage($route)
		  ↓
	(kgs-content/pages/... JSON source OR upstream sync source)
		  ↓
	CMSCache::set/get (GoogleDriveCache-backed)
		  ↓
	Router renders:
		header → layout → components → footer
	
2. Responsibility boundaries (NO CROSSING)

	A. ContentCMS (SOURCE OF TRUTH)

		File: kgs-core/cms/ContentCMS.php

		ONLY allowed to:

		Resolve a route → page structure
		Read from:
		kgs-content/pages/*
		OR upstream sync layer (if enabled)
		Return:
		[
		  'meta' => [...],
		  'components' => [...]
		]

		NOT allowed to:

		cache anything
		know Google Drive cache exists
		render UI
		mutate routing

	B. CMSCache (APPLICATION CACHE LAYER)

		File: kgs-core/cms/CMSCache.php

		ONLY allowed to:

		Store/retrieve normalized CMS pages
		Wrap GoogleDriveCache
		Provide TTL behavior via underlying cache system

		It does NOT:

		fetch content from disk
		decide page structure
		interpret CMS rules

		It is ONLY:

		ContentCMS → CMSCache → GoogleDriveCache → filesystem

	C. GoogleDriveCache (INFRASTRUCTURE CACHE ENGINE)

		File: kgs-core/google/GoogleDriveCache.php

		ONLY allowed to:

		Read/write JSON cache files
		Enforce schema
		TTL checks
		File IO only

		It must NEVER:

		know what a “page” is
		know routes exist
		know CMS structure

		It only stores:

		group + key → raw JSON payload

	D. Router (APPLICATION CONTROLLER)

		File: kgs-core/Router.php

		ONLY allowed to:

		resolve route
		call CMS service
		pass result into layout system
		render components

		It must NOT:

		talk to cache directly
		talk to Google services
		interpret CMS storage formats

	E. app/components (VIEW LAYER)

		ONLY allowed to:

		render UI from:
		$type + $data

		NO business logic
		NO caching
		NO routing

3. Data model (single source of truth)

	Every page MUST resolve to:

	[
	  'meta' => [
		'title' => '',
		'description' => '',
		'layout' => ''
	  ],
	  'components' => [
		[
		  'type' => 'hero',
		  'data' => []
		]
	  ]
	]

	Nothing else is required for rendering.

4. Cache model (correct mental model)

	ContentCMS (truth)
		  ↓
	CMSCache (application cache)
		  ↓
	GoogleDriveCache (filesystem persistence)
		  ↓
	kgs-cache/google/cms_pages_cache/*.json

	Cache is:

	replaceable
	disposable
	never authoritative

5. NO-CONFUSION RULE SET (critical)

	RULE 1 — ONE SOURCE OF TRUTH

		Only this is authoritative:

		kgs-content/pages

		Cache is never truth.

	RULE 2 — ONE DIRECTION OF DEPENDENCY

		Allowed:

		ContentCMS → CMSCache → GoogleDriveCache

		NOT allowed:

		GoogleDriveCache → CMSCache → ContentCMS
		Router → GoogleDriveCache
		Components → CMSCache

	RULE 3 — NO “DUAL CMS” LOGIC

		You currently have:

		ContentCMS (active)
		CMSCache (supporting)

		This is OK.

		What is NOT OK:

		any second CMS loader
		any parallel “DriveCMS vs ContentCMS” split

	RULE 4 — CACHE NEVER DEFINES STRUCTURE

		Cache may contain:

		meta
		components

		Cache must NEVER contain:

		routing rules
		file system structure
		transformation logic

	RULE 5 — ROUTER IS PURE ORCHESTRATION

		Router does:

		route → page → render

		Nothing else.

	RULE 6 — COMPONENTS ARE DUMB RENDERERS

		They receive:

		$data

		They do NOT:

		fetch CMS
		fetch cache
		resolve routes

6. What your current structure now correctly achieves

	From your updated tree:

	✔ ContentCMS exists (correct consolidation point)
	✔ kgs-content/pages exists (correct source-of-truth layer)
	✔ CMSCache is isolated (good)
	✔ GoogleDriveCache is infrastructure-only (correct)
	✔ Router is still in control of rendering pipeline (correct)

7. Final sanity check (what NOT to change right now)

	Do NOT touch:

	GoogleDriveCache (already correct abstraction level)
	folder structure under kgs-content/pages
	component system
	Router flow

	Only evolve:

	ContentCMS internals (if needed)
	CMSCache behavior (only performance-related changes)

8. One-line mental model

	ContentCMS = truth
	CMSCache = speed
	GoogleDriveCache = disk
	Router = conductor
	Components = paint

Below is a **single, stable architecture definition + enforcement script** based on your current tree and the corrected responsibility model.

This is designed to stop the exact category of confusion you’ve been hitting: duplicate responsibilities, cache/content overlap, and silent architectural drift.

---

# 1. Final “Clean Architecture” (what you actually have now)

## Layer 1 — Public Entry

**`public/index.php` → Router**

* Accepts request
* Resolves route
* Delegates to CMS service
* NEVER touches cache or content directly

---

## Layer 2 — Routing

**`kgs-core/Router.php`**

Responsibilities:

* Normalize route
* Apply aliases
* Call CMS service only

Allowed:

* `ContentCMSService::load($route)`

Not allowed:

* Reading files
* Calling cache directly
* Rendering components

---

## Layer 3 — CMS Orchestration

**`kgs-core/services/ContentCMSService.php`**

Responsibilities:

* Decide source of page data:

  * cache first (`CMSCache`)
  * fallback (`ContentCMS`)
* Return normalized page array

Allowed:

* `CMSCache`
* `ContentCMS::loadPage()`

Not allowed:

* Rendering
* HTTP output
* Direct filesystem access to page folders

---

## Layer 4 — Content Source of Truth

**`kgs-core/cms/ContentCMS.php`**

Responsibilities:

* Load canonical page data from:

  ```
  kgs-content/pages/{route}/meta.json
  kgs-content/pages/{route}/components.json
  ```
* Return raw structured array

Not allowed:

* Cache writes
* Google logic
* HTTP logic

---

## Layer 5 — Cache Layer

### `kgs-core/cms/CMSCache.php`

Responsibilities:

* Store/retrieve **final rendered page arrays**
* Wrap `GoogleDriveCache`

Allowed:

* `GoogleDriveCache`

Not allowed:

* ContentCMS calls
* Routing logic

---

## Layer 6 — Storage Engine

### `kgs-core/google/GoogleDriveCache.php`

Responsibilities:

* Generic JSON file cache system

Knows NOTHING about:

* CMS
* Pages
* Routes
* Content structure

Only:

* set/get/delete JSON blobs

---

## Layer 7 — Content Storage

### `kgs-content/pages/...`

This is your:

> SINGLE SOURCE OF TRUTH (IMPORTANT)

Contains:

* `meta.json`
* `components.json`

No PHP logic.

---

## Layer 8 — Renderer

### `app/components/renderer.php`

Responsibilities:

* Convert components → HTML

Allowed input:

* `components[]`

Not allowed:

* cache
* content loading
* routing

---

# 3. “No Confusion Ruleset” (hard rules)

These are the rules that prevent future rewrites:

## RULE 1 — Only ONE source of truth

```
kgs-content/pages = ONLY canonical data
```

## RULE 2 — Cache is NEVER authoritative

Cache may:

* disappear
* be deleted
* be stale

System must still work.

---

## RULE 3 — CMSCache NEVER loads content

CMSCache:

* stores final page arrays only

It does NOT:

* build pages
* read JSON files
* talk to GoogleDriveCache directly except storage

---

## RULE 4 — ContentCMS is PURE LOADER

ContentCMS:

* reads disk only
* no cache
* no Google
* no router

---

## RULE 5 — Router is dumb

Router:

* resolves route
* calls service
* stops

---

## RULE 6 — Renderer is dumb

Renderer:

* receives data
* outputs HTML
* no loading logic

---

## RULE 7 — GoogleDriveCache is generic infrastructure

It does NOT know:

* “pages”
* “CMS”
* “routes”

Only:

* group + key + payload

# 4. Minimal “Architecture Test Script” (IMPORTANT)

Create:

```
tools/scripts/architecture-test.php
```

### Purpose:

Fails build if boundaries are violated.

---

## ARCH TEST CODE

```php
<?php

$failures = [];

function scanFile($file, $rules)
{
    global $failures;

    $content = file_get_contents($file);

    foreach ($rules as $ruleName => $patterns) {
        foreach ($patterns as $pattern) {
            if (strpos($content, $pattern) !== false) {
                $failures[] = "$file violates $ruleName ($pattern)";
            }
        }
    }
}

/*
|---------------------------------------------
| RULE SET
|---------------------------------------------
*/

$rules = [

    'Router must not access cache or content' => [
        'GoogleDriveCache::',
        'CMSCache::set',
        'ContentCMS::loadFromFile',
        'file_get_contents(kgs-content'
    ],

    'ContentCMS must not use cache' => [
        'CMSCache::',
        'GoogleDriveCache::get',
        'GoogleDriveCache::set'
    ],

    'CMSCache must not load files directly' => [
        'file_get_contents',
        'kgs-content'
    ],

    'Renderer must not load CMS or cache' => [
        'ContentCMS::',
        'CMSCache::',
        'GoogleDriveCache::'
    ],
];

/*
|---------------------------------------------
| FILES TO CHECK
|---------------------------------------------
*/

$files = new RecursiveIteratorIterator(
    new RecursiveDirectoryIterator(__DIR__ . '/../../kgs-core')
);

foreach ($files as $file) {

    if (!$file->isFile() || $file->getExtension() !== 'php') {
        continue;
    }

    scanFile($file->getPathname(), $rules);
}

/*
|---------------------------------------------
| RESULT
|---------------------------------------------
*/

if (!empty($failures)) {
    echo "ARCHITECTURE VIOLATIONS FOUND:\n\n";
    foreach ($failures as $f) {
        echo $f . "\n";
    }
    exit(1);
}

echo "Architecture OK\n";
exit(0);
```
